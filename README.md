An automated, robust R-based pipeline designed for the analysis of EMS (Ethyl methanesulfonate) mutagenesis experiments in *Medicago polymorpha*. This toolkit provides end-to-end analytical solutions ranging from toxicological dose-response modeling (LD50 estimation) to rigorous statistical enrichment of phenotypic mutation spectra.

This repository contains two core analytical modules that ensure statistical rigor, false-discovery rate (FDR) control, and publication-quality (SCI standard) data visualization.

## 📦 Dependencies

Ensure you have the following R packages installed before running the scripts:

install.packages(c("dplyr", "tidyr", "ggplot2", "drc", "readr", "broom", "tools", "forcats", "ggsci", "scales", "stringr", "conflicted"))

## 📜 Script 1: `01_Dose_Response_Physiology.R`

**Description:** This script evaluates the physiological damage and optimal mutation concentration thresholds following EMS treatment. It utilizes non-linear regression and quadratic modeling to assess seedling survival and mutation rates.

### 🔑 Key Features:
- **Log-logistic Dose-Response Modeling:** Fits a robust LL.4 model via the `drc` package to calculate LD10, LD50, and LD90 values. Includes forced-fallback Loess smoothing if the underlying data fails to converge.
- **Quadratic Optimization:** Automatically fits a quadratic regression to determine the theoretical optimal EMS concentration for maximizing the M2 mutation rate.
- **Pure `ggplot2` Correlation Matrix:** Generates a highly customizable, WCAG 2.1 colorblind-friendly correlation matrix with dynamic text-color inversion for maximum readability.
- **Faceted Trait Trends:** Visualizes various physiological traits across EMS gradients with automatically calculated error bars.

### 📥 Input:
- `EMS_M2_MP2603V2_Sheet1.csv` (Wide-format dataset containing concentrations, trait means, and standard deviations).

### 📤 Key Outputs:
- `Fig1_LD_Model_Fit.png`: Dose-response curve with 95% confidence intervals.
- `Fig2_Mutation_Optimization.png`: Quadratic fit pinpointing optimal mutagenesis concentration.
- `Fig3_Multi_Trait_Correlation.png`: High-contrast Pearson correlation matrix.
- `02_LD_10_50_90_with_CI.csv`: Calculated lethal doses and standard errors.

## 📜 Script 2: `02_Mutation_Spectrum_Enrichment.R`

**Description:** This script provides a stringent statistical framework for identifying mutation hotspots and analyzing phenotypic diversity among M2 plant lines, correcting for sample size variations and false positive inflation.

### 🔑 Key Features:
- **Data Pre-flight Audit:** Automatically sanitizes column names via regex, parses mixed-format frequency data (e.g., `7.83%` vs `0.0783`), and filters out underpowered lineages (n < 5) to prevent statistical artifacts.
- **FDR-Corrected Exact Binomial Test:** Evaluates the global mutation burden of each line against the population baseline, controlling for multiple comparisons via Benjamini-Hochberg (BH).
- **Standardized Residuals Heatmap:** Employs a Monte Carlo-simulated Chi-square test (10,000 permutations) to extract standardized residuals. Hotspots are explicitly marked only if they pass strict FDR correction ($q < 0.05$).
- **Overlap Defense Visualization:** Dynamically scales plot widths based on the number of lineages and intelligently hides text labels in stacked bar charts if the proportion is < 5%.

### 📥 Input:
- `EMS_M2_MP2603V2_Sheet3.csv` (Long-format phenotype tally dataset recording mutants per plant line and category).

### 📤 Key Outputs:
- `00_Data_Audit_Log.txt` & `00_Session_Info.txt`: Complete logs ensuring full reproducibility.
- `Fig1_Spectrum_Lollipop.png`: Overall phenotypic mutation spectrum ordered by frequency.
- `Fig2_Enrichment_Heatmap_FDR.png`: Hotspot heatmap with rigorous statistical annotations.
- `Fig3_Diversity_StackedBar.png`: Proportional stacked bar charts showing the diversity of mutations within specific *M. polymorpha* lines.

## 🚀 Usage

1. Clone the repository to your local machine.
2. Place your raw data files (`Sheet1.csv` and `Sheet3.csv`) in the root directory of the project.
3. Open the scripts in RStudio.
4. Source the scripts sequentially. Each script will automatically generate a uniquely timestamped output directory (e.g., `EMS_Analysis_20260328_153000/`) containing all processed CSVs, statistical tables, and 300 DPI PNG figures.
